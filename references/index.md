# Landing Page模板参考（2025设计哲学）

## ⚠️ 核心原则
**`template/` 下的 15 个 HTML 模板是每种风格的最佳实践标杆——CSS 技巧、动效实现、响应式布局、交互模式均经过验证。AI 生成时应以此为质量基线对齐或超越，同时根据具体产品灵活调整，充分参考各模板文档的"美学重点"章节。**

## ⚠️ 用户交互提示
**与用户对话时，使用风格名称（如"经典Hero+Features"、"便当盒式"等）推荐和描述风格。template_id和表格中的模板编号仅作为内部参考，用于脚本调用和智能体内部推理，不应向用户暴露。**

## 2025三大核心设计哲学

### 物理质感回归（Return of Physics）
摒弃死板的扁平化，在数字世界中重建"重量感"和"价值感"。
- **关键词**：液态玻璃（Liquid Glass）、体积光（Volumetric Lighting）、磨砂透视（Frosted Glass）、真实材质渲染
- **代表风格**：深色沉浸式、电影感/硬件流、浅色虹彩液态玻璃、数字工坊/静谧艺廊

### 网格化与模块化（Bento & Modular）
受AI Agent和系统化思维影响，信息被封装在独立的"容器"中。
- **关键词**：Bento Grid（便当盒网格）、卡片式布局、非线性浏览
- **代表风格**：多彩层级版、动态商务风、便当盒式

### 展示即交互（Show, Don't Tell）
用户要看"活"的东西，建立即刻信任（Instant Trust）。
- **关键词**：实时代码终端（Live Terminal）、可交互演示（Interactive Demo）、流式生成动画
- **代表风格**：双列布局+视频、趣味连接型、代码原生型

## 美学增强规范

所有着陆页在基础结构完成后，必须实施 **六大美学增强要素**：

1. **固定导航** — 毛玻璃/半透明固定 Navbar，滚动后切换样式
2. **移动端汉堡菜单** — ≤768px 折叠为三线汉堡按钮，点击展开下拉
3. **滚动揭示** — IntersectionObserver 驱动的渐入动画
4. **Hero 装饰** — 浮动几何/有机形状，增加空间层次
5. **图片动态加载** — 支持运行时图片注入 + CSS 渐变降级（图片由生成阶段根据产品内容动态规划）
6. **JS 交互增强** — 平滑滚动 + Navbar 滚动检测 + 汉堡切换 + 可选计数器

- **设计原则**：`.cursor/rules/aesthetic-enhancement.mdc`（Cursor 自动加载）
- **实现指南**：`references/aesthetic-enhancement-guide.md`（完整代码模板）

## 用户指令到Config字段的映射

### 产品信息
产品名称/叫什么 → product.name
标语/口号/slogan → product.tagline
描述/简介 → product.description
价格/多少钱 → product.price
图片/产品图 → product.image

### 功能管理
增加功能/添加功能 → features数组append
第N个功能改成 → features[N-1]修改
删除第N个功能 → features数组splice(N-1, 1)
修改功能描述 → features[].description修改

### CTA调整
按钮/按钮文字 → cta.button_text
标题/主标题 → cta.title
副标题/说明 → cta.subtitle

### 评价管理
增加评价/添加评价 → mock_data.testimonials数组append
删除第N条评价 → mock_data.testimonials数组splice(N-1, 1)

### 样式调整
背景色/背景颜色 → theme.background_color
主色/主色调 → theme.primary_color
文字色/文字颜色 → theme.text_color
