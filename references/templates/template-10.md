# template-10: 动态商务风

## ⚠️ 核心原则
**本文档的"美学重点"章节是生成时的首要参考。HTML/CSS技术细节仅作为排版结构和命名规范参考，生成时应以美学高度和视觉冲击力为首要目标，不拘泥于代码细节。**


**设计理念**：
引入"几何切割"(Geometric Slicing)的设计语言，打破传统网页水平分割的沉闷感，带来锐利、进取的商务气质。页面不再是方方正正的矩形堆叠，而是通过倾斜的分割线(Angled Dividers)来连接不同板块。这种设计在视觉上极具动感，通常用于强调"速度"、"突破"或"现代科技"。

**结构组成**：
1. Slanted Hero区域（深色背景+底部斜切）
   - 背景：深海军蓝(Navy Blue #0F172A)纯色背景
   - 布局：典型Split Layout（左文右图）
   - **关键特征**：底部边缘不是水平的，而是向右下倾斜的（clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%)）
   - 斜切效果：直接打破平衡，引导视线强制向下流动
   - 样式：文字反白，按钮采用高亮的蓝色(Electric Blue #3B82F6)形成对比

2. Structured Zig-Zag区域（白底+线条感）
   - 回归白底，但在排版上更强调线条感
   - **水平分割线**：每个标题或段落下方都有明显的水平分割线(3px solid #3B82F6)
   - 布局：经典的"图文交错"模式
   - 视觉感：比前几版更加严谨、规整，有"说明书"或"技术文档"的理性感
   - Divider Lines：使用细线条分割内容，保持结构清晰

3. Minimal Feature Grid（极简功能网格）
   - 3列×2行网格
   - 非常干净：没有卡片背景，没有边框，纯粹依靠Icon+Text的留白来区分
   - 视觉上非常轻量，适合信息密度不高的展示
   - 图标：淡蓝色背景(#E0F2FE)，电光蓝色图标

4. Card-Based Testimonials（卡片式证言）
   - 3列×2行卡片
   - 样式：卡片带有轻微的阴影，背景为纯白
   - 悬浮在灰白色(#F8FAFC)的大背景上
   - 微弱的层次感让评价显得很精致

5. Clean FAQ（清爽问答）
   - 左对齐的列表布局
   - 排版：使用细线条作为分割(2px solid #E2E8F0)
   - 保持Structured Zig-Zag区域的设计语言一致性

6. Pre-Footer（淡蓝背景块）
   - Light Blue Block(#E0F2FE)
   - 没有使用深色压轴，而是用了很轻的淡蓝色
   - 给用户一种轻松的结束感

7. Footer（极简多列链接）
   - 背景：回归纯白或极淡的灰
   - 结构：简单的多列链接（产品、解决方案、资源、公司）
   - 底部：版权信息和分割线

**核心特点**：
- 🔪 **几何切割**：倾斜分割线打破水平沉闷感
- 📐 **斜切角度**：Hero底部斜切，引导视线向下流动
- 📏 **线条感强**：水平分割线贯穿整个设计
- 💼 **商务气质**：深蓝配色+锐利线条，专业进取
- ⚡ **动感十足**：倾斜设计强调"速度"和"突破"
- 🎯 **理性结构**：类似技术文档的严谨规整

**适用场景**：
- 企业B2B软件（需要专业形象）
- 专业工具（强调严谨和可靠）
- 金融机构（需要信任感）
- 咨询服务（展示专业能力）
- 强调"速度"、"突破"的产品

**配置要点**：

**必填字段**：
```yaml
product:
  name: 产品名称
  tagline: 一句话价值主张
```

**Hero配置（可选）**：
```yaml
hero:
  image_url: 首屏右侧图片URL（可选）
```

**Structured Sections配置（新增）**：
```yaml
structured_sections:
  - title: 第一个区块标题
    description: 详细描述...
    image_url: 插图URL（可选）
  - title: 第二个区块标题
    description: 详细描述...
    image_url: 插图URL（可选）
  - title: 第三个区块标题
    description: 详细描述...
    image_url: 插图URL（可选）
```

**功能网格**：
```yaml
features:
  - title: 功能1
    description: 简短描述
  - title: 功能2
    description: 简短描述
  # ... 共6个功能
```

**可选字段**：
```yaml
testimonials:
  - name: 用户姓名
    role: 身份/职业
    content: 评价内容
    avatar_url: 头像URL
  # ... 共3-6个评价

faq:
  - question: 常见问题1
    answer: 详细答案
  # ... 建议3-6个问题

cta:
  title: Pre-Footer标题
  subtitle: 补充说明
  button_text: 按钮文字
```

**视觉风格建议**：
- 🎨 配色方案：
  - 深海军蓝：#0F172A（Hero背景）
  - 电光蓝：#3B82F6（强调色、按钮、分割线）
  - 淡蓝色：#E0F2FE（图标背景、Pre-Footer）
  - 灰白色：#F8FAFC（Testimonials背景）
  - 纯白色：#ffffff（主要内容区）
- 📐 斜切角度：Hero底部85%（clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%)）
- 📏 水平分割线：3px solid #3B82F6（Structured标题底部）
- 💫 圆角：小圆角（4-8px），商务风格
- 🖼️ 阴影：轻微阴影，精致层次感
- 📝 字体：干净的无衬线字体，标题加粗
- 🎯 对比：深色背景上的反白文字，淡蓝色背景上的蓝色图标
- 📐 布局：左右分屏、图文交错、网格布局

**技术实现要点**：
- ✅ **斜切实现**：使用`clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%)`裁剪Hero容器
- ✅ **响应式处理**：移动端可以移除斜切或调整角度
- ✅ **水平分割线**：标题下方使用3px solid #3B82F6
- ✅ **极简网格**：无卡片、无边框，纯留白+图标区分
- ✅ **配色统一**：电光蓝(#3B82F6)作为强调色贯穿全站

**使用建议**：
- ✅ 适合企业B2B软件（需要专业形象）
- ✅ 适合专业工具（强调严谨和可靠）
- ✅ 适合金融机构（需要信任感）
- ✅ 适合咨询服务（展示专业能力）
- ✅ 适合强调"速度"、"突破"的产品
- ❌ 不适合创意类产品（推荐template-06/07）
- ❌ 不适合极简休闲风格（推荐template-05）

## 美学重点
- 📐 **几何切割设计**：Hero斜切角、硬朗线条，营造专业商务感
- 💙 **电光蓝主色调**：#3B82F6作为强调色贯穿全站，统一视觉语言
- 📏 **水平分割线**：3px实线分割，强化商务风格
- 🧮 **极简网格布局**：无卡片无边框，纯留白+图标，信息层次清晰
- 💫 **精致阴影**：轻微阴影营造层次感，克制而专业
- 🎯 **严谨商务美学**：小圆角、硬朗线条，适合B2B和专业服务场景
