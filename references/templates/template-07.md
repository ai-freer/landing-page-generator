# template-07: 多彩层级版

## ⚠️ 核心原则
**本文档的"美学重点"章节是生成时的首要参考。HTML/CSS技术细节仅作为排版结构和命名规范参考，生成时应以美学高度和视觉冲击力为首要目标，不拘泥于代码细节。**


**设计理念**：
通过大面积色块分区来创造视觉节奏和层次感，形成"分幕剧"般的浏览体验。第一幕（蓝）开场、第二幕（白）叙事、第三幕（蓝）功能展示、第四幕（绿）用户信任与转化。这种设计通过色彩变化有效降低阅读疲劳，特别适合功能点较多、需要用户长时间浏览的产品。

**结构组成**：
1. Split Hero区域（左右分屏，淡蓝色背景）
   - 左侧：H1标题、副标题、CTA按钮
   - 右侧：核心插画/产品截图
   - 全宽淡蓝色渐变背景：#e8f4ff → #f5faff

2. Zig-Zag Flow区域（白底，3个部分）
   - 左右交错布局（Why/What/How叙事）
   - 纯白背景：将核心叙事连成一个整体
   - 视觉引导：从左到右、从上到下的流动

3. Feature Grid区域（3×2网格，淡蓝色背景）
   - 全宽色块区：淡蓝色渐变背景（#e8f4ff → #f0f7ff）
   - 白色卡片悬浮在蓝色背景上（ elevation + 阴影）
   - 每个卡片：数字图标+功能标题+描述
   - 作用：通过色彩变化打破白底单调，重新抓取注意力

4. Testimonials区域（3列卡片，淡绿色背景）
   - 背景色切换为淡绿色：#e8fff0 → #f0fff7
   - 色彩心理学：绿色代表"安全"、"通过"、"信任"
   - 白色卡片悬浮在淡绿背景上，形成清晰的层级和立体感

5. FAQ区域（白底）
   - 回归冷静的白底，方便专注阅读文字
   - 折叠式问答列表

6. Pre-Footer CTA区域（淡绿色背景）
   - 呼应Testimonials的绿色，利用建立起的"信任感"颜色引导转化
   - 居中布局：标题、副标题、CTA按钮

7. Footer（深色背景）
   - 深色背景：#1a1a1a
   - 品牌名称、标语、社交链接

**核心特点**：
- 🎨 色块分区：大面积背景色块创造视觉节奏
- 🎭 分幕式体验：蓝→白→蓝→绿的颜色流动
- 💡 视觉引导强：色彩变化引导用户浏览
- 📊 功能密集：适合功能点较多的产品
- 🔁 降低疲劳：色彩变化有效降低阅读疲劳
- ⬆️ 层级清晰：有色背景上的白色卡片形成 elevation

**适用场景**：
- 现代SaaS平台（功能点较多）
- 科技产品（需要展示大量功能）
- 企业级软件（需要长时间浏览）
- 复杂工具（需要分层次展示）

**配置要点**：

**必填字段**：
```yaml
product:
  name: 产品名称
  tagline: 一句话价值主张
```

**Hero配置（新增）**：
```yaml
hero:
  image_url: 首屏右侧插图URL（可选）
```

**核心配置（zigzag_sections）**：
```yaml
zigzag_sections:
  - title: 第一部分标题（如：为什么选择我们）
    description: 详细描述...
    image_url: 插图URL（可选）
  - title: 第二部分标题（如：核心功能）
    description: 详细描述...
    image_url: 插图URL（可选）
  - title: 第三部分标题（如：如何开始）
    description: 详细描述...
    image_url: 插图URL（可选）
```

**功能网格**：
```yaml
features:
  - title: 功能1
    description: 简短描述
  - title: 功能2
    description: 简短描述
  # ... 共6个功能
```

**可选字段**：
```yaml
testimonials:
  - name: 用户姓名
    role: 身份/职业
    content: 评价内容
    avatar_url: 头像URL
  # ... 共3个评价

faq:
  - question: 常见问题1
    answer: 详细答案

cta:
  title: 底部CTA标题
  subtitle: 补充说明
  button_text: 按钮文字

footer:
  tagline: 品牌标语
  social_links:
    - name: Twitter
      url: https://twitter.com/...
```

**视觉风格建议**：
- 🎨 配色方案：
  - 淡蓝色：#e8f4ff → #f5faff（Hero、Features）
  - 淡绿色：#e8fff0 → #f0fff7（Testimonials、CTA）
  - 白色：#ffffff（ZigZag、FAQ）
  - 深色：#1a1a1a（Footer）
- 💜 主色调：紫色渐变（#667eea → #764ba2）用于CTA按钮和图标
- 📝 字体：干净的无衬线字体，标题加粗
- 🖼️ 图片：高质量的产品截图或插画
- ⚪ 圆角：24px大圆角（Hero图）、16px中圆角（卡片）
- 💫 阴影：柔和的阴影效果，营造 elevation
- 📐 布局：左右分屏Hero，3×2网格Features
- 🎯 对比：有色背景上的白色卡片形成强对比

**使用建议**：
- ✅ 适合功能点较多（6个以上）的产品
- ✅ 适合需要用户长时间浏览的页面
- ✅ 内容丰富，需要层次感的产品
- ✅ 现代、科技感强的产品
- ❌ 不适合极简风格产品（推荐template-05）
- ❌ 不适合情感化叙事为主的产品（推荐template-06）

## 美学重点
- 🎨 **多彩色块分区**：淡蓝、淡绿、白色交替，视觉节奏感强
- 📐 **色块层级布局**：每个区域用不同色块区分，信息层次清晰
- 💜 **紫色渐变CTA**：统一的紫色视觉语言，引导转化
- ⚪ **大圆角卡片**：Hero图24px、卡片16px，现代友好
- 💫 **柔和阴影**：营造elevation，增强空间感
- 🌟 **丰富内容层次**：适合功能点多的产品，信息展示有序不混乱
